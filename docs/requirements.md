# 機能要件定義

## 目的

パチンコの大当り後にガチャ動画を表示し、その後ランダム景品を獲得できる収集要素を追加する。

## 要件

- 大当り倍率が10以上のとき、既存のガチャ動画を再生する。
- ガチャ動画の表示終了後、`assets/keihin` 配下の3Dデータからランダムに景品を表示する。
- 景品収集枠としてバッジ10個、カード10個を用意する。
- 景品枠はパチンコ台の左右に配置し、獲得済み枠が視覚的に埋まる。
- 収集状況は `localStorage` に保存し、リロード後も維持する。
- 初回アクセス時にクライアント側で端末IDを発行し、パスワードなしで同一ブラウザの収集状態を復元する。
- 景品獲得時は3Dプレビューを表示し、一定時間後または閉じる操作で非表示にできる。
- SOUND ONの状態でユーザー操作により音声が有効化されたら、`assets/bgm` 配下のMP3をBGMとしてループ再生する。

## 対象外

- サーバーサイドAPIによる景品管理。
- `assets/keihin` フォルダの動的ディレクトリ走査。
- 景品のレアリティや交換機能。
