# 実装中のつまづきと解決方法

## `assets/keihin` の動的列挙

静的HTMLをブラウザで配信する構成では、JavaScriptからローカルディレクトリを直接列挙できない。そのため、今回存在を確認できたGLBファイルを `PRIZE_ASSETS` に明示定義した。

## ガチャ動画終了タイミング

既存実装は `gachaMovieFrames` を毎フレーム減らして動画表示を制御していた。動画の `ended` イベントとフレームカウント終了のどちらでも景品表示へ進めるよう、`finishGachaMovie()` に終了処理を集約した。

## 3Dモデル表示

単体HTMLでGLBを扱いやすくするため、`<model-viewer>` を利用した。読み込みに失敗した場合でも景品名と収集枠更新は動作する。

## パスワードなしの保存

ログイン機構を持たないため、同じ端末かどうかは厳密にはブラウザの `localStorage` に依存する。初回アクセス時に `ikapachinkoDeviceId` を発行し、収集データに同梱することで、将来サーバー同期へ移す場合にも移行しやすい形式にした。

## BGMの自動再生制限

ブラウザはユーザー操作前の音声再生をブロックするため、ページ読み込み直後にはBGMを開始しない。既存のSOUND解除処理に合わせて、クリック、キー入力、SOUNDボタン操作でAudioContextが `running` になった後に `bgm_main` をループ再生する。
